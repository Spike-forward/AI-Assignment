習作一：自動搜集圖像數據集與初步處理

習作簡介

本習作旨在透過編程實作，加強學生在使用編程實現自動化圖像數據集搜集和初步處理方面的技能。學生實踐自動化網絡搜索、圖片下載和預處理圖像，為後續分析做好準備。

此習作佔本科目總分的 15%。

習作要求

●      使用指定的關鍵字在網上搜尋圖像（例如：Google 圖像搜索）

●      需有 3000 至 5000 個圖像

●      收集圖像的網址（src）和替代文字（alt）

●      將圖像下載到本地資料夾

●      將圖像調整大小並置中裁剪為不超過 500x500 像素，編碼為 JPEG (50-80 quality)

●      如果圖像大小超過 50KB，繼續調整為更小的尺寸，直至符合要求

●      列出收集的圖像數量

●      程式可用 Typescript，Javascript，或者 Python 實現

●      適當使用函數和類來組織代碼，使其清晰易於理解和維護

建議使用編程庫

●      Playwright: 用於自動驅動瀏覽器

●      Sharp / tensorflowjs / pillow / opencv: 用於縮放、裁剪和轉換圖像格式

●      quick-erd: 用於設計和維護數據庫

●      better-sqlite3-proxy: 用於存取數據庫

數據集主題

每位學生會被分配到一個圖像數據集主題 i，其中 i 是學生 ID 除以 10 的餘數。

假設學生 ID 是 28361933，那麼：

  1. 計算 28361933 mod  10 得到 3
  2. 學生會被分配到圖像數據集 3 的主題

以下是十個主題：

  1. 人類/人形動漫角色，各類型的性別和職業
  2. 各類型的貓咪品種
  1. 不同風格的建築物
  2. 城市中的交通工具
  3. 各種不同品種的狗
  4. 多樣化的自然風景
  5. 不同時代的時尚服飾
  6. 各種運動中的運動員
  7. 世界各地的特色美食
  8. 不同情感表達的人物

●      收集圖像的網址和替代文字：15 分

○      正確收集並記錄圖像的 src 和 alt 信息。

圖像下載和處理（50 分）

●      下載和儲檔：20 分

○      成功下載圖像 3000 至 5000 個圖像。

○      儲存在相應的文件夾。

○      列出收集的圖像數量。

●      調整和裁剪：20 分

○      正確調整大小並置中裁剪為不超過 500x500 像素。

●      重新編碼和尺寸控制：10 分

○      圖像重新編碼為 JPEG，質量設置在 50-80，並確保大小不超過 50KB。

自動化和代碼質量（20 分）

●      自動化程度：10 分

○      自動化處理數據搜集和下載的能力。

●      程式碼結構和註釋：10 分

○      代碼的模組化、清晰度和註釋的完整性。

 

總分：100 分，佔本科目總分的 15%。

具體評分標準

搜索和收集（30 分）

關鍵字搜索和結果相關性：15 分

●      優良：使用精確關鍵字，獲得90%以上相關圖像。

●      滿意：使用較好關鍵字，獲得70%-89%相關圖像。

●      有待改善：使用不當關鍵字，獲得少於70%相關圖像。

收集圖像的網址和替代文字：15 分

●      優良：所有圖像的 src 和 alt 信息完整、正確。

●      滿意：大部分圖像的 src 和 alt 信息正確，少量缺失。

●      有待改善：許多圖像的 src 和 alt 信息缺失或錯誤。

圖像下載和處理（50 分）

下載和儲檔：20 分

●      優良：成功下載 3000 至 5000 個圖像，並正確儲存在相應的文件夾。

●      滿意：大部分圖像下載成功，數量接近要求。

●      有待改善：多個圖像未下載或數量不足。

調整和裁剪：20 分

●      優良：所有圖像正確調整和裁剪為不超過 500x500 像素。

●      滿意：大部分圖像正確調整和裁剪，少數有誤。

●      有待改善：多數圖像未正確調整或裁剪。

重新編碼和尺寸控制：10 分

●      優良：所有圖像重新編碼為 JPEG，質量 50-80，大小不超過 50KB。

●      滿意：大部分圖像符合要求，少數超過 50KB。

●      有待改善：多數圖像未達到質量或尺寸要求。

自動化和代碼質量（20 分）

自動化程度：10 分

●      優良：完全自動化，無需人工干預。

●      滿意：部分自動化，少量人工干預。

●      有待改善：需要大量人工操作。

程式碼結構和註釋：10 分

●      優良：代碼結構清晰，模組化良好，註釋詳盡。

●      滿意：代碼結構基本清晰，有基本註釋。

●      有待改善：代碼結構混亂，缺乏註釋。

提交格式

●      將源代碼、數據庫、和下載了的圖像壓縮成 ZIP 檔案。

●      確保所有更改都已提交並註釋清楚。

●      包含程序輸出的截圖或輸出日誌文件。
